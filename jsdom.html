JSDOM

In javascript, the document object provided by the browser can be used to manipolate the document object model and update viewer content via javascript. In the following example DOM manipulation through selectors and If-Elese statemet is explain using a basic "To-Do- List"

To-Do List - DOM Manipolation

A todo list interface is created in HTML.

    Example:

    JavaScript Code:

                    
                  // Function to add a new task
                  function addTask() {
                    const taskInput = document.getElementById("taskInput");
                    const taskList = document.getElementById("taskList");
                    const taskText = taskInput.value.trim();

                    if (taskText !== "") {
                      const newTask = document.createElement("li");
                      newTask.textContent = taskText;

                      taskList.appendChild(newTask);
                      taskInput.value = "";
                    } else {
                      alert("Please enter a task!");
                    }
                  }

    Code Explanation

    • An input field allows users to add new tasks, which are displayed as list items within an unordered list.

    • A JavaScript function, addTask(), is defined to handle adding new tasks to the list.

    • document.getElementById("taskInput") function is used to retrieve the HTML element with id "taskInput". This function is part of the Document Object Model (DOM) API, which allows javascript to interact with HTML elements. It searches the entire HTML document for an element with the specified id and returns a reference to that element. The returned value, taskInput, is assigned to a constant variable using the const keyword. Once assigned, taskInput represents the input element where users can enter new tasks.

    • document.getElementById("taskList") is similar to the previous line and retrieves the HTML element with the id "taskList". The returned element represnts the unordered list where tasks are displayed is assigned to the constant variable "taskList" This allows javascript to manipulate the task list, such as adding new tasks or updating existing ones.

    • The value entered into the task input field is extracted and stored in the variable taskText. taskInput.value retrieves the current value of the input element (taskInput). .trim() is a string method that removes any leading and trailing whitespace from the value. The trimmed value is then assigned to the constant variable taskText. This variable holds the text of the task entered by the user, with leading and trailing whitespace removed for consistency.

    • The if statement initiates a conditional statement. It checks whether the taskText variable is not equal to an empty string (" "). The condition taskText !== " " evaluates to true if the user has entered some text for the task and false if the input is empty. If the condition is true, indicating that the user has entered a task, the code inside the curly braces {} following the if statement will be executed. Otherwise, if the condition is false, the code inside the curly braces will be skipped.

    • Next, inside the if block, a new list item element is created using the document.createElement() method. The method document.createElement("li") creates a new HTML element with the specified tag name, in this case, a list item. The newly created list item element is assigned to a constant variable named newTask using the const keyword. This variable holds a reference to the newly created list item element, which will represent the new task in the to-do list.

    • After creating the new list item element, textContent property of an HTML element is used to set the text content to the value stored in the taskText variable. By assigning taskText to newTask.textContent, the text entered by the user for the new task is added as the content of the newly created list item. This line effectively populates the new list item with the text of the task entered by the user, making it visible within the to-do list.

    • taskList.appendChild(newTask) appends the newly created list item (newTask) to the task list (taskList). The appendChild() method is a part of the DOM API, specifically for manipulating the structure of HTML documents. It adds the specified node (newTask in this case) as the last child of the node (taskList) on which the method is called. In this context, newTask, representing the new task entered by the user, is added as a new item in the to-do list represented by taskList. After this line executes, the new task becomes part of the displayed list, visible to the user on the web page.

    • taskInput.value = "" resets the value of the task input field (taskInput) to an empty string (""). By setting taskInput.value to "", the input field is cleared after a task is successfully added to the list. This action provides a clean slate for the user to enter a new task without having to manually clear the previous input. After this line executes, the input field is ready for the user to enter another task, improving the user experience by streamlining the task entry process.

    • The else keyword introduces an alternative block of code to be executed if the condition specified in the preceding if statement evaluates to false. In this context, the else block serves as the alternative action when the user attempts to add a task but leaves the input field empty. If the condition taskText !== "" evaluates to false, indicating that the input field is empty, the code inside the else block will be executed.

    • Inside the else block, alert("Please enter a task!") triggers a JavaScript alert dialog with the message "Please enter a task!". The alert() function is a built-in JavaScript function used to display a dialog box with a specified message to the user. In this case, the alert informs the user that they must enter a task before adding it to the to-do list. The execution of this line interrupts the flow of the script and displays the alert message in the user's browser window. After the user dismisses the alert by clicking "OK", the script continues executing, without adding an empty task to the list or performing any further actions related to task addition.
